Midworld Development Roadmap

Welcome to the Midworld development roadmap. This document briefly describes where the Midworld project has been and where it's going.

Last Modified: $Date: 2002-06-21 04:46:47 $
Revision: $Revision: 1.1 $

Iteration 1 (28 June 2002)

  • Moveable Main Character
    The main character should be moveable by the player. The player should always face the mouse cursor. The player should be able to rotate using the mouse, move forward, move backward, strafe left, and strafe right using the keyboard.
  • AI Framework
    The framework for the AI system should be defined, but not yet integrated into the game.
  • Collision Detection Framework
    The framework for the Collision Detection system should be defined, but not yet integrated into the game.
  • Random Enemies
    A set number of enemies will begin the game that will walk about in random directions. They shall be represented as cubes.
  • Physics Engine (Linear)
    A basic physics engine should be implemented that at least handles linear movement and forces.
  • Basic Weapon Functionality
    The player should be able to fire off shots of cubical bullets in the direction he/she is facing.
  • Visible Terrain
    There should be some type of terrain that is visible so that the player is not moving along a dark (or bright) mass.
  • Integration with GameKernel
    Midworld should be run as a GameKernel application.

Iteration 2 (5 July 2002)

  • Model Loading
    Cubes should be replaced with models loaded from disk, using a scenegraph such as OpenSG or OpenSceneGraph.
  • Collision Detection (Basic)
    Basic collision detection should be integrated so that we can do some initial bounds collision tests.
  • Physics Engine (Torque)
    The physics engine should support torque forces.
  • Maps Loaded From Disk
    Maps should be loaded from disk from some sort of file format.
