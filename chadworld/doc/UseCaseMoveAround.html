Problem B, iii.  Fully dressed use cases
Use Case:  move around

 

Primary actor:  Player

Stakeholders and Interests:

-         Player: Wants to move around in the world.

Preconditions

-         The player is connected to the server and data is flowing back and forth without a problem

-         The player has a specific dynamics set (their position, velocity, heading, etc)

-         The world is "running" in that the players dynamics are being applied through time

-         The world on the client and server are synchronized.

Success Guarantee (post conditions):

-         The player's dynamics are changed on both the client and server.

 

Main Success Scenario (Happy path!):

1.      The player decides he wants to change their dynamics in the world (their velocity, in general)

2.      The player push a button on the keyboard, or moves the mouse

3.      The client software forwards that request to the server

4.      The server applies the request to the world on its own side

5.      The server sends a request to the client, which then applies it (the request) to itself (the client).  (Since the worlds are the same on both sides, the same result will occur on the local copy as did on the server's copy.)

Extensions (or alternate flows):

*a. at anytime the network connection dies.

1.      A java exception is thrown on the client and server side.

a.       On the server side, that connection link is removed from the pool of connections

b.      On the client side, the program is exited, with a message stating that the connection failed.

2a. The player presses a button that does not correspond with any commands designed int the game

            1. Nothing happens!

4a.  Due to the world's state, the requested command cannot be carried out, such as a wall or other player being in the way.

1.      The server calculates a new modification to the dynamics set based on the desired transformation, and what is possible.

2.      the server sends this new transform, in lieu of sending the original request back to the client.
