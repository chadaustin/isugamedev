Problem B, iii.  Fully dressed use cases
Use Case:  start up.

 

Primary actor:  Player

Stakeholders and Interests:

-         Player: Wants to connect to the world to look around

Preconditions

-         The server is running on this, or another computer.

-         The two computers are on the same network, able to contact each other.

Success Guarantee (post conditions):

-         The player is deposited in the world with a specific set of dynamics.

-         A connection is established so that information can pass between the two computers

 

Main Success Scenario (Happy path!):

1.      The Player loads the program

2.      the client program prompts the user to enter the address of a server

3.      The user enters the address of the server

4.      The client connects to the address specified

5.      The server program modifies the world to add a representation of the client into it.

6.      the server sends the client a copy of the world.

Extensions (or alternate flows):

4.      If the computer can’t connect to the server specified

a.       The program prints a message indicating that it can’t, and the process goes back to step 2

Special Requirements:

-         None

Technology and data variations list:

-         none

Open issues:

-         none
